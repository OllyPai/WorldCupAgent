# 世界杯竞赛播报助手前端执行 Plan

## 1. 任务来源与职责理解

1. 本 Plan 基于文档  
   `proposal-report.pdf`

2. 项目正式名称  
   基于大模型的工具调用智能体——世界杯竞赛播报助手

3. 项目技术背景  
   a. 后端方向：`Python + LangChain / LangGraph + DeepSeek API + FastAPI`  
   b. 数据侧：`SQLite（缓存）+ JSON`  
   c. 前端侧：负责 Web 界面与可视化展示

4. 文档中对你职责的明确描述  
   a. `Web 前端 + 可视化`  
   b. `负责对话界面开发`  
   c. `负责工具调用过程可视化`  
   d. `负责前后端联调`

5. 这意味着你的任务重点不是只做一个首页  
   a. 首页只是入口  
   b. 真正核心是把“用户提问 - Agent 调用工具 - 返回结果”完整展示出来  
   c. 页面必须体现这是一个 `tool-calling agent`，不是普通聊天页

## 2. 前端目标重定义

1. 前端总目标  
   做一个能展示世界杯问答流程的 Web 前端，让老师和组员一眼看到：
   a. 用户可以自然语言提问  
   b. 系统会调用工具查询世界杯数据  
   c. 查询过程是透明可见的  
   d. 结果可以用文本、表格、卡片三种形式展示

2. 前端必须承接的课程功能点
   a. 自然语言输入界面  
   b. 查询结果展示  
   c. 工具调用过程可视化  
   d. 多轮对话展示  
   e. 错误提示与空结果提示

3. 前端成果标准  
   a. 能独立演示首页  
   b. 能演示问答页  
   c. 能展示一次完整工具调用链  
   d. 能接后端真实接口  
   e. 视觉上具备赛事系统和 AI 系统特征

## 3. 前端页面规划

1. `首页 Home`
   a. 项目标题与简介  
   b. 今日比赛 / 热门比赛  
   c. 快捷问题入口  
   d. Agent 能力说明  
   e. 进入查询页入口

2. `智能查询页 Chat / Query`
   a. 用户输入问题  
   b. 展示用户消息与系统回答  
   c. 支持多轮对话记录  
   d. 支持快捷问题点击填充  
   e. 支持加载中、失败、无结果状态

3. `工具调用过程面板 Trace Panel`
   a. 展示识别意图  
   b. 展示调用工具名  
   c. 展示工具参数  
   d. 展示工具返回摘要  
   e. 展示执行步骤时间线

4. `结果展示区 Result Panel`
   a. 文本回答  
   b. 表格展示  
   c. 卡片展示  
   d. 不同问题类型切换不同展示方式

5. `比赛详情页 Match Detail`
   a. 比赛双方  
   b. 比分与状态  
   c. 进球信息  
   d. 比赛阶段  
   e. 重要事件时间线

## 4. 页面优先级与开发顺序

1. 第一阶段  
   完成首页静态展示  
   目标：先把项目门面立住

2. 第二阶段  
   完成智能查询页静态结构  
   目标：把“用户输入 + 回答输出 + Trace 面板”搭起来

3. 第三阶段  
   用 mock 数据打通前端演示流程  
   目标：即使后端没接好，也能演示主要页面

4. 第四阶段  
   接入真实后端接口  
   目标：完成前后端联调

5. 第五阶段  
   补齐异常状态、空状态、加载状态  
   目标：满足课程展示与测试要求

   d. `返回结构化赛事结果接口`
