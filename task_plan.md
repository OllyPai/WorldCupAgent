# 任务计划：WorldCupAgent 课程项目

## 目标

完成一个可演示、可测试、可协作交付的世界杯工具调用智能体，同时确保郑梓湃能够解释并修改 Agent 调用流程、模块接口和测试方法。

## 当前阶段

阶段 5：Web 联调（FastAPI 适配层已启动）

## 完成标准

- 支持直接回答、单工具、多工具、多轮对话和错误处理；
- 接入赛程、球员数据、比赛详情三个工具；
- 提供稳定的 `answer / tool_calls / error` 统一入口；
- Web 界面能传入历史并展示回答、调用过程和错误；
- 系统测试不少于 10 条，包含 Mock 自动测试和真实 API 冒烟测试；
- README、依赖、环境变量示例和课程文档能够支持新环境运行与答辩；
- 核心实现、测试和个人贡献具有清晰的 Git 提交记录。

## 范围约束

- 当前场景固定为世界杯竞赛播报助手；
- 必做工具固定为赛程、球员数据、比赛详情；
- 不提前实现积分榜、射手榜、流式输出等扩展功能；
- 当前 Agent 工作实现使用 LangChain `create_agent` 和 DeepSeek；
- Web 技术架构在统一入口稳定前保持待定；
- 自动测试不请求真实模型，真实 API 只做少量冒烟验证；
- AI 处理语法、样板和重复工作；用户重点掌握调用流程、接口和调试。

## 已做决策

| 决策 | 理由 |
|------|------|
| 使用 Python 3.12 虚拟环境 | 隔离项目依赖并与当前安装一致 |
| 当前使用 LangChain `create_agent` | 学习现成 Agent 工具，减少自建循环成本 |
| 使用 DeepSeek 作为真实模型 | 已完成配置和真实调用验证 |
| Mock 自动测试与真实 API 冒烟测试分离 | 保证测试稳定、快速且不持续消耗 API |
| 前端只依赖统一入口 | 避免 Web 框架影响 Agent 内部实现 |
| 规划文件放在仓库根目录 | 支持跨会话恢复和项目级进度控制 |
| 工具型问题最终回答由代码格式化 | 真实模型会做额外推断；代码格式化能保证只展示工具返回字段 |
| React 前端通过 FastAPI 调用 Agent | 前端不能直接 import Python 函数，需要 HTTP 边界 |
| API 层将内部 `error` 状态映射为前端 `failed` | 对齐前端契约，不改变 Agent 内部结构 |

## 遇到的错误

| 错误 | 尝试次数 | 解决方案/状态 |
|------|---------|---------------|
| fish 无法加载 `.venv/bin/activate` | 1 | 改用 `.venv/bin/activate.fish` |
| 未激活环境时找不到 `python` | 1 | 命令中使用 `.venv/bin/python` 或先激活环境 |
| 测试先于入口实现，导入 `chat_with_agent` 失败 | 1 | 实现统一入口后测试通过 |
| 模型最终回答加入工具未返回的事实 | 2 | 仅靠 Prompt 仍会推断，已改为工具型问题由代码格式化最终回答 |
| 沙盒禁止 `git merge` 写入 `.git/ORIG_HEAD.lock` | 1 | 使用授权后的同一条 `git merge --no-commit --no-ff origin/feat/tools-data` 成功 |
| 当前虚拟环境缺少 `streamlit` 命令 | 1 | 判定前端方向应改为 React + FastAPI，不继续使用 Streamlit |
