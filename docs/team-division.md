# 团队分工文档

## 项目概述

- **选题**：基于大模型的工具调用智能体——世界杯竞赛播报助手
- **团队成员**：3 人
- **技术栈**：Python + LangChain/LangGraph + DeepSeek + FastAPI（后端）+ 前端自选框架（C 同学决定）

## 角色 A：Agent 核心 + LLM 集成（你）

### 核心职责

1. **Agent 推理循环搭建**
   - 使用 LangChain/LangGraph 构建 Agent
   - 实现「理解需求 → 制定计划 → 选择工具 → 调用工具 → 观察结果 → 生成回答」完整流程
   - 处理多步骤任务分解（如「巴西队赛程 + 当前积分榜排名」需要串行调用多个工具）

2. **Prompt 设计与调优**
   - System prompt：定义 Agent 角色、行为规范、工具使用策略
   - Few-shot examples：提供典型问答示例，提升工具选择准确率
   - 错误恢复 prompt：工具调用失败时的重试/降级策略

3. **工具注册与集成**
   - 将角色 B 开发的工具注册到 LangChain Tool 体系
   - 定义每个工具的 name、description、parameters schema
   - 确保工具描述足够清晰，大模型能准确选择

4. **多轮对话与上下文管理**
   - 记录对话历史，关联上下文
   - 处理用户追问（不需要重复背景信息）

5. **核心流程联调**
   - 与角色 B 联调工具接口
   - 与角色 C 联调前端接口（SSE 事件格式定义）
   - 提供 FastAPI 接口层，将 Agent 包装为 REST API

### 技术要点

- LangChain Agent (ReAct / Tool-calling agent)
- LangGraph 状态图（如果任务需要多步骤规划）
- DeepSeek Chat API（function calling 模式）
- FastAPI + SSE（Server-Sent Events）流式输出
- 工具调用步骤通过 SSE 事件实时推送前端

### 交付物

- Agent 核心代码（`agent/` 模块）
- Prompt 设计文档
- Agent 单元测试（mock 工具调用）
- 系统流程图

## 角色 B：工具开发 + 数据层

### 核心职责

1. **世界杯数据源接入**
   - 调研并接入世界杯数据 API（如 API-Football、Football-data.org 等）
   - 处理 API 认证、速率限制
   - 数据缓存策略（避免重复请求、应对 API 不可用）

2. **工具实现（≥3 个）**
   - **赛程查询工具**：按球队/日期/阶段查询比赛安排
   - **实时比分工具**：查询比赛结果、进球详情
   - **积分榜工具**：查询小组或淘汰赛阶段排名
   - 可选扩展：射手榜、球队阵容、历史交锋记录

3. **工具封装**
   - 每个工具封装为标准 LangChain Tool 格式
   - 明确定义：名称、功能描述、输入参数（Pydantic schema）、返回格式
   - 错误处理：API 超时、数据缺失时的优雅降级

4. **数据格式化**
   - 将 API 返回的原始 JSON 格式化为 Agent 易理解的结构
   - 确保返回内容适合前端渲染（便于角色 C 展示为表格/卡片）

### 技术要点

- `requests` / `httpx` HTTP 调用
- JSON 数据解析与清洗
- LangChain `@tool` 装饰器
- Pydantic BaseModel 参数定义
- 可选：SQLite 缓存 API 响应

### 交付物

- 工具模块代码（`tools/` 模块）
- API 调研报告（选择了哪个数据源、为什么）
- 工具测试用例（手动验证数据准确性）

## 角色 C：Web 前端 + 可视化

### 核心职责

1. **对话界面**
   - 用户输入框与对话消息流
   - AI 回答渲染（支持 Markdown、表格）
   - 美观的界面设计（自选前端框架：React / Vue / 原生 HTML+JS 均可）

2. **工具调用过程可视化（答辩加分项）**
   - 实时展示 Agent 调用了哪个工具
   - 显示工具输入参数和返回结果
   - Agent 思考过程透明化（「正在查询巴西队赛程…→ 查到 3 场比赛 → 正在查询积分榜…」）
   - 状态指示器（加载中、成功、失败）

3. **结果展示**
   - 比分卡片样式
   - 积分榜表格
   - 赛程时间线
   - 可选：简单图表（柱状图对比射手榜等）

4. **与 Agent 后端交互**
   - 通过 SSE 接收 Agent 流式输出和工具调用事件
   - 处理流式输出逐字展示
   - 错误提示（API 失败时给出友好提示）

5. **与 Agent 后端联调**
   - 与角色 A 对齐 SSE 事件格式
   - 确定前后端数据契约

### 技术要点

- 自选前端框架（React/Vue/原生均可）
- EventSource / fetch SSE 流式消费
- Markdown 渲染
- 表格与卡片 UI 组件
- 可选：ECharts / Chart.js 数据图表

### 交付物

- 前端代码（`frontend/` 目录）
- UI 截图（用于文档）
- 用户交互流程说明
- 前后端接口文档（与 A 协作）
