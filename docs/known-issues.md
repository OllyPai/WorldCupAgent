
当前已注册 7 个查询工具：

| 工具 | 能力 |

| `query_top10_scorers` | 查询射手榜前十 |

## 3. 需要继续讨论的问题

### 3.1 真实 LLM 工具选择稳定性

工具本身已经存在，自动测试也覆盖了工具注册、格式化和部分业务返回。但真实模型在自然语言问题下是否稳定选择正确工具，还需要端到端冒烟验证。

建议保留为讨论项，而不是立即继续堆新功能。可用以下问题测试：

1. 请查询巴西队赛程
2. 请查询梅西的世界杯进球数据
3. 请查询阿根廷和佛得角的比赛详情
4. 阿根廷进球最多的球员是谁
5. 谁是进球最多的球员
6. 谁是扑救最多的门将
7. 中国和佛得角分别有什么比赛
8. 中国和佛得角的比赛详情
9. 帮我写一段 C++ 排序代码
10. `???###`

判断标准：

- 能查数据库的问题应调用对应工具；
- 明显超范围问题应返回系统能力边界；
- “分别查询”和“两队比赛详情”不能混淆；
- 回答只能基于工具结果，不补充数据库外事实。

## 4. 当前不作为问题处理的事项

以下事项当前不进入问题清单：

- 本地 SQLite 数据库是课程演示数据，不接官方实时 API；
- 首页存在前端示例数据，用于展示产品形态；
- 本地未跟踪文件和历史分支属于工作区管理，不影响当前功能判断；
- Git 历史中曾出现过集成分支选择问题，但当前 `main` 已包含可运行版本。

## 5. 下一步建议

当前优先级：

1. 先用真实前后端跑一轮固定演示问题，确认 Agent 工具选择稳定性；
2. 前端根据 `result_payload.mode` 逐步补充结构化展示；
3. 对暂不支持的 `mode`，前端降级展示 `answer`。
